Does PCA Actually Help a Housing Model?

It is easy to treat principal component analysis as a respectable preprocessing step and much harder to prove that compressing a housing feature table buys enough predictive stability to justify the interpretability you lose on the way.

What the notebook really says

Baseline linear model `R2 0.65`
PCA with all components `R2 0.65`
PCA with 6 components `R2 0.66`
Variance retained at 6 `68.4%`

The score lift is real but narrow. The editorial question is whether a one-point improvement in explained variance is worth replacing understandable housing features with six abstract axes.

Left panel / Original feature space

Readable, correlated, operationally legible

Area, bedrooms, bathrooms, stories, parking, furnishing status, and two engineered ratios tell a story a housing analyst can actually repeat to another person. You can argue about whether the model overweights area or whether areaperbedroom is doing double duty, but at least the argument remains grounded in the property itself.

The notebook confirms that this space is not clean. The maximum feature correlation is `0.8056`, coming from a predictable overlap between area and areaperbedroom. That is exactly the sort of redundancy PCA is designed to compress.

Right panel / PCA space

Cleaner geometry, weaker explanation

Once the standardized training set is rotated into principal components, the new axes are effectively uncorrelated. The notebook's post-PCA correlation matrix has off-diagonal values that are numerically near zero. From a geometric standpoint, that is tidy. From a business standpoint, it is instantly harder to explain.

PC1 can absorb area, parking, and bathroom scale; PC2 can pull in bedroom tradeoffs and areaperbedroom; but none of that reads like the language buyers, brokers, or lenders actually use when they discuss a house.

Core tension

PCA earns a cleaner modeling surface here. It does not earn an automatic promotion to “best practice.”

Original side / business meaning
PCA side / statistical convenience

The notebook becomes interesting only when you stop asking whether PCA is valid and start asking whether it is worth the trade.

A lot of PCA writeups make the same mistake. They treat dimensionality reduction as a sign of methodological maturity, as if “we applied PCA” were already a conclusion. That is not how this housing case reads to me. The notebook gives us a cleaner question: once we have a 15-column feature table and a plain linear model that already works decently, what exactly are we buying by rotating the space?

The answer is not “better prediction” in any dramatic sense. The baseline linear regression on standardized original predictors lands at mean squared error `0.35` and `R2 0.65`. Running regression on the full PCA-transformed space also lands at `0.35` and `0.65`. That should not surprise anyone. If you keep all components, PCA is mostly a rotation. You gain orthogonality. You do not meaningfully simplify the problem.

The only moment PCA starts behaving like a substantive choice is when the notebook cuts down to `6` components. Then the model moves to mean squared error `0.33` and `R2 0.66`. That is a modest improvement, not a revolution. Still, it is enough to justify a serious comparison because the dimensionality drop is real: six components instead of fifteen predictors, with cumulative explained variance around `68.4%`.

What it means in practice

The strongest technical argument for PCA in this file is not speed. It is decorrelation.

The dataset has `545` rows and `16` columns total, with `price` as the target and `15` predictors after feature selection. Most of them are easy to recognize: structural counts like bedrooms and bathrooms, amenity flags like air conditioning, and derived ratios like areaperbedroom and bbratio. The housing problem itself is ordinary. The tension comes from feature overlap.

The notebook measures that overlap explicitly. The maximum non-diagonal correlation is `0.8056104195904636`. That is high enough to create unstable coefficient stories in a linear model, especially when engineered ratios and raw size variables coexist. If you have ever tried to explain why a bedroom coefficient flips or shrinks after adding area-normalized features, you already know why PCA is tempting here.

After standardization, PCA rotates the training data into orthogonal components. The resulting principal-component correlation matrix has off-diagonal values on the order of `1e-15`, which is functionally zero. That part is not cosmetic. It means the regression downstream is operating on dimensions that no longer compete with one another for the same sliver of signal.

There is a real gain in that cleanup. If the only consumer of the model were another algorithm, I would be more sympathetic to PCA as a default preprocessing step. Cleaner geometry often helps when the alternative is a feature space full of redundant directions.

But housing models are rarely consumed only by machines. Someone usually wants to know why a predicted price moved, which feature family matters most, whether a premium came from square footage or furnishing quality, or whether the model is overreacting to rare amenity combinations. PCA makes those questions harder, not impossible, and “harder” is a real cost when the business is already capable of reasoning in named variables.

That is why I read the notebook's slight performance lift with caution. Decorrelating the space is valuable. Making the explanation layer fuzzier is also real. The model improvement has to be large enough to pay for that fuzziness. Here it is present, but barely.

Why all-component PCA changes so little
Why reduced PCA changes the argument

The notebook accidentally teaches the most important PCA lesson: reduction is the point, not the rotation.

If you keep all components, PCA is mostly a re-expression of the same information. That is why the all-component regression looks indistinguishable from the baseline. Anyone who expected a dramatic jump there was hoping for magic. PCA is not magic. It cannot create predictive information that was not already in the feature table.

The six-component model is different because it actually throws information away. That makes the notebook more interesting, not less, because now there is a defensible trade: some variance is discarded, redundancy is reduced, and the downstream regression edges upward to `R2 0.66`. The case for PCA lives in that narrow band.

What I would not do is oversell the gain. A movement from `0.65` to `0.66` is not a mandate. It is a prompt to ask a stricter question: in the environment where this model will be used, is a one-point gain in explained variance more valuable than keeping the model legible in terms of bedrooms, bathrooms, area, and furnishing status? In an automated ranking pipeline, maybe yes. In a decision memo that needs feature-level accountability, probably not.

from sklearn.decomposition import PCA
from sklearn.linear_model import LinearRegression

pca = PCA(n_components=6, random_state=100)
Xtrain_reduced = pca.fit_transform(Xtrain)
Xtest_reduced = pca.transform(Xtest)

model = LinearRegression()
model.fit(Xtrain_reduced, ytrain)
y_pred = model.predict(Xtest_reduced)  # smaller space, weaker direct meaning

The code is simple. The implication is not. Six unnamed components are easier for the regression to digest and harder for a human to interrogate. That is the entire editorial conflict condensed to eight lines.

Figure B / Use-decision matrix

My read

If the audience for this model is going to ask “why this price?”, I would keep the original feature space and accept the correlated inputs. If the audience only cares about compact batch prediction, the six-component PCA version is defensible. I would not present the latter as an unqualified improvement.

Bottom line
Action

PCA helps this housing model a little. The explanation layer pays most of the bill.

The notebook answers its own question more honestly than most PCA demos do. Using every principal component changes almost nothing in predictive performance, which proves that PCA is not a free accuracy hack. Reducing to six components trims the space and nudges fit from `R2 0.65` to `0.66`, which is a respectable result but not a sweeping mandate.

So my recommendation is conditional, not devotional. Keep the original standardized features when the model needs to remain interpretable in real-estate language. Use the reduced PCA pipeline only when geometric cleanup and compactness matter more than direct explanation. The right conclusion here is not “PCA is useful.” The right conclusion is narrower: PCA is useful when you can afford to stop speaking in the language of houses.
